# お手軽タスク管理アプリ v1 仕様書

## 1. バージョン概要

v1では、デスクトップ常駐型のシンプルなタスク管理アプリを実装する。

目的は、ユーザーがすばやくタスクを追加し、作業完了時に作業時間を記録できる最小実用版を完成させることである。

## 2. コンセプト

本アプリは、昔のWindowsのイルカアイコンのように、ユーザーの作業中に常駐し、必要なときだけワンタッチで開ける軽量タスク管理アプリである。

重厚なプロジェクト管理ではなく、以下の体験を重視する。

- すぐ開ける
- すぐ追加できる
- すぐ並び替えられる
- すぐ完了できる
- 作業時間を簡単に残せる
- オフラインでも使える

## 3. 想定ユーザー

- 日々の細かい作業を管理したい個人ユーザー
- タスク管理アプリを開くこと自体が面倒なユーザー
- 作業時間ログを簡単に残したいユーザー
- 日報や振り返りの材料を残したいユーザー

## 4. v1で実装する機能

| 機能 | 内容 |
|---|---|
| アプリ常駐 | OS起動中にバックグラウンドで動作する |
| ワンタッチ表示 | タスクトレイまたはフローティングアイコンから開ける |
| タスク追加 | タイトルのみでタスクを追加できる |
| 完了チェック | チェックボックスで完了状態にできる |
| 作業時間入力 | 完了時に作業時間を分単位で入力できる |
| ドラッグ並び替え | 未完了タスクをドラッグで並び替えできる |
| ローカル保存 | SQLiteまたはJSONでローカル保存する |
| 完了済み表示 | 完了済みタスクを下部に表示する |
| 今日の集計 | 当日の完了件数と作業時間合計を表示する |
| タスク削除 | 不要なタスクを削除できる |

## 5. 画面仕様

### 5.1 メイン画面イメージ

```text
🐬 今日のタスク

[ タスクを入力して Enter ]

□ 仕様書を確認する
□ メールに返信する
□ 会議メモを整理する

--- 完了済み ---

☑ 資料をダウンロードする　10分

今日の完了：1件 / 10分
```

### 5.2 画面の基本方針

- 小さなウィンドウで表示できること
- 入力欄は常に分かりやすい位置に置くこと
- 未完了タスクを優先して表示すること
- 完了済みタスクは下部に表示すること
- 完了済みタスクは薄い表示、または取り消し線で表示すること

## 6. タスク追加仕様

### 6.1 入力方法

メイン画面上部の入力欄にタスクタイトルを入力し、Enterキーで追加する。

```text
[ タスクを入力して Enter ]
```

### 6.2 追加時のルール

- 入力欄が空の場合は追加しない
- タスクタイトルのみで追加できる
- 追加後、入力欄を空にする
- 新規タスクは未完了状態で追加する
- 新規タスクは未完了リストの最上部に追加する

### 6.3 入力必須項目

v1では、タスク追加時の必須項目はタイトルのみとする。

作業時間はタスク完了時に入力する。

## 7. タスク項目仕様

### 7.1 タスクデータ項目

| 項目 | 型 | 必須 | 説明 |
|---|---|---|---|
| id | string | 必須 | タスク一意ID |
| title | string | 必須 | タスク名 |
| completed | boolean | 必須 | 完了状態 |
| order | number | 必須 | 表示順 |
| spent_minutes | number | 任意 | 完了までにかかった時間 |
| created_at | datetime | 必須 | 作成日時 |
| completed_at | datetime | 任意 | 完了日時 |
| updated_at | datetime | 必須 | 更新日時 |

### 7.2 JSON例

```json
{
  "id": "task_001",
  "title": "仕様書を確認する",
  "completed": false,
  "order": 1,
  "spent_minutes": null,
  "created_at": "2026-05-28T09:00:00+09:00",
  "completed_at": null,
  "updated_at": "2026-05-28T09:00:00+09:00"
}
```

## 8. 完了チェック仕様

### 8.1 完了操作

ユーザーがチェックボックスをクリックすると、作業時間入力ダイアログを表示する。

### 8.2 作業時間入力ダイアログ

```text
このタスクにかかった時間は？

[ 30 ] 分

[保存] [スキップ]
```

### 8.3 入力ルール

- 分単位で入力する
- 数値のみ入力可能
- 空欄の場合は未入力として保存する
- 0分も許可する
- 保存後、タスクを完了済みにする
- completed_at に完了日時を保存する

### 8.4 スキップ時の挙動

- spent_minutes は null のまま保存する
- タスクは完了済みにする
- completed_at は保存する

### 8.5 完了取り消し

完了済みタスクのチェックを外した場合、以下の処理を行う。

- completed を false に戻す
- completed_at を null に戻す
- spent_minutes は保持する

作業時間を保持する理由は、誤操作で完了解除した場合でも記録を失わないようにするためである。

## 9. ドラッグ並び替え仕様

### 9.1 対象

未完了タスクをドラッグして順番変更できる。

### 9.2 挙動

- タスクを上下にドラッグ移動できる
- 並び替え後、order を更新する
- 並び順は自動保存する
- アプリ再起動後も並び順を保持する

### 9.3 完了済みタスクの扱い

v1では、完了済みタスクは未完了タスクの下に表示する。

```text
未完了タスク
---
完了済みタスク
```

## 10. 常駐アイコン仕様

### 10.1 表示方式

v1では、以下のいずれかを実装する。

- タスクトレイ常駐アイコン
- 画面端のフローティングアイコン

推奨は、タスクトレイ常駐アイコンである。

### 10.2 アイコンクリック時の挙動

- 非表示状態ならメイン画面を表示する
- 表示状態ならメイン画面を前面に出す
- 最小化されている場合は復元する

### 10.3 アイコンの方向性

- 小さな相棒キャラクター風
- タスク数バッジを将来的に表示可能
- クリックしやすく、邪魔にならないこと

## 11. 保存仕様

### 11.1 保存方式

v1ではローカル保存を基本とする。

候補は以下。

```text
SQLite
または
JSONファイル
```

推奨は SQLite。

理由は以下。

- タスク履歴が増えても扱いやすい
- 検索や集計に拡張しやすい
- 将来的な日報出力や履歴管理と相性が良い

### 11.2 保存タイミング

以下の操作時に自動保存する。

- タスク追加
- タスク編集
- タスク完了
- 作業時間入力
- 並び替え
- 完了取り消し
- タスク削除

## 12. 今日の集計仕様

### 12.1 表示項目

メイン画面下部に、当日の簡易集計を表示する。

```text
今日の完了：5件 / 2時間30分
```

### 12.2 集計対象

- completed_at が今日のタスク
- completed が true のタスク

作業時間合計は、spent_minutes が入力されているタスクのみを対象とする。

### 12.3 表示形式

| 合計分 | 表示 |
|---:|---|
| 45 | 45分 |
| 90 | 1時間30分 |
| 150 | 2時間30分 |

## 13. タスク削除仕様

### 13.1 削除方法

各タスクに削除ボタンを用意する。

```text
□ タスク名    [×]
```

### 13.2 削除確認

誤操作防止のため、削除時は確認を表示する。

```text
このタスクを削除しますか？

[削除] [キャンセル]
```

### 13.3 削除方式

v1では物理削除または論理削除のどちらでもよい。

推奨は論理削除。

将来的な履歴復元や集計の拡張に対応しやすいため。

## 14. 非対象機能

v1では以下を実装しない。

- チーム共有
- ガントチャート
- カレンダー連携
- 複雑なタグ管理
- 複数プロジェクト管理
- コメント機能
- AI自動分類
- ブラウザ拡張連携
- 右クリックメニュー連携

## 16. v1 完了条件

v1は、以下を満たした時点で完成とする。

- アプリを起動できる
- タスクトレイまたはアイコンから開ける
- タスクをタイトルだけで追加できる
- タスクを完了できる
- 完了時に作業時間を入力できる
- タスクをドラッグで並び替えできる
- アプリ再起動後もタスクが残っている
- 当日の完了件数と作業時間合計が表示される
- 不要なタスクを削除できる

## 17. v1 MVP定義

最初に作るべきMVPは以下とする。

```text
・Windowsで起動できる
・タスクトレイに常駐する
・クリックでタスクリストが開く
・タスクをタイトルだけで追加できる
・チェックで完了できる
・完了時に作業時間を入力できる
・タスクをドラッグで並び替えできる
・SQLiteに保存される
・再起動後もデータが残る
・今日の完了数と作業時間が表示される
```
