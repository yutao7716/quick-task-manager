# お手軽タスク管理アプリ v2 仕様書

## 1. バージョン概要

v2では、他のシンプルなタスク管理アプリとの差別化要素として、選択文章からタスクを追加する機能を実装する。

メール、Webページ、チャット、資料などの文章を選択し、右クリックメニューやショートカットからタスクに追加できるようにする。

## 2. v2の位置付け

v1は最小実用版、v1.5は実用強化版である。

v2は、作業中に発生したタスクを逃さず捕まえるための差別化版である。

## 3. v2の中心コンセプト

```text
文章を見つける
↓
選択する
↓
右クリックまたはショートカット
↓
タスク化する
```

タスク管理アプリを開いて入力するのではなく、作業中の文脈から直接タスクを作成できることを重視する。

## 4. v2で追加する主な機能

| 機能 | 内容 |
|---|---|
| 右クリックメニュー追加 | 選択文章をタスクとして追加する |
| クリップボードから追加 | コピーした文章をタスク化する |
| 元アプリ名保存 | Chrome、Outlookなどのアプリ名を保存する |
| 元URL保存 | ブラウザで選択した場合、URLを保存する |
| 選択テキスト保存 | タスク本文とは別に元文章を保存する |
| ブラウザ拡張連携 | Webページ上の文章をタスク化する |
| タスク詳細画面 | 元テキスト、URL、メモを確認できる |
| 日報・1on1メモ連携 | 作業ログをMarkdownで整理出力する |
| インポート/エクスポート | JSON、CSV、Markdownで出力する |

## 5. 前提条件

v2は、v1およびv1.5の以下機能が実装済みであることを前提とする。

- アプリ常駐
- ワンタッチ表示
- タスク追加
- 完了チェック
- 作業時間入力
- ドラッグ並び替え
- ローカル保存
- ショートカット起動
- タスク検索
- 完了履歴
- 日報コピー
- 簡易メモ

## 6. 右クリックメニュー追加仕様

### 6.1 目的

任意の文章を選択し、右クリックメニューからタスクとして追加できるようにする。

### 6.2 操作イメージ

```text
文章を選択
↓
右クリック
↓
「タスクに追加」
↓
タスクリストに追加
```

### 6.3 作成されるタスク

選択文章の先頭部分をタスクタイトルにする。

選択文章例。

```text
来週までに営業資料の修正版を作成してください。
```

作成タスク例。

```text
□ 来週までに営業資料の修正版を作成してください。
```

### 6.4 タイトル長制限

タスクタイトルは長くなりすぎないようにする。

推奨は以下。

```text
最大100文字
```

100文字を超える場合は省略する。

例。

```text
来週までに営業資料の修正版を作成してください。詳細については...
```

元の全文は source_text に保存する。

### 6.5 右クリック連携の対象

優先順位は以下。

1. ブラウザ上の選択テキスト
2. メールアプリ上の選択テキスト
3. チャットアプリ上の選択テキスト
4. PDFやドキュメント上の選択テキスト

v2初期では、ブラウザを最優先にする。

## 7. クリップボードから追加仕様

### 7.1 目的

右クリックメニュー連携が難しい環境でも、コピーした文章を簡単にタスク化できるようにする。

### 7.2 操作例

```text
文章をコピー
↓
アプリを開く
↓
[クリップボードから追加]
```

またはショートカット。

Windows。

```text
Ctrl + Shift + V
```

Mac。

```text
Command + Shift + V
```

### 7.3 追加仕様

- クリップボードの文字列を取得する
- 空の場合は追加しない
- 先頭100文字をタイトルにする
- 全文を source_text に保存する
- source_type は clipboard とする

### 7.4 重複防止

直前に同じクリップボード内容からタスク追加している場合、重複追加を防止する。

確認例。

```text
同じ内容のタスクが直前に追加されています。
もう一度追加しますか？

[追加] [キャンセル]
```

## 8. 元情報保存仕様

### 8.1 目的

あとから「このタスクは何の文章から作ったのか」を確認できるようにする。

### 8.2 保存項目

| 項目 | 型 | 説明 |
|---|---|---|
| source_text | string | 選択した元文章 |
| source_app | string | 元アプリ名 |
| source_url | string | 元URL |
| source_title | string | Webページタイトル |
| source_type | string | web / app / clipboard |
| source_created_at | datetime | タスク化した日時 |

### 8.3 保存例

```json
{
  "title": "営業資料の修正版を作成してください",
  "source_text": "来週までに営業資料の修正版を作成してください。特に価格表と導入事例を更新してください。",
  "source_app": "Google Chrome",
  "source_url": "https://example.com/message/123",
  "source_title": "営業チームチャット",
  "source_type": "web",
  "source_created_at": "2026-05-28T10:30:00+09:00"
}
```

## 9. ブラウザ拡張連携仕様

### 9.1 対象ブラウザ

優先順位は以下。

```text
1. Chrome
2. Edge
3. Firefox
```

v2初期では Chrome を最優先とする。

### 9.2 ブラウザ拡張の機能

- 選択文章を取得する
- 現在のページURLを取得する
- ページタイトルを取得する
- ローカルアプリへ送信する
- タスク追加完了を通知する

### 9.3 ブラウザ右クリックメニュー

ブラウザ上で文章を選択した場合、右クリックメニューに以下を表示する。

```text
タスクに追加
```

### 9.4 ローカルアプリとの連携方法

候補は以下。

```text
カスタムURLスキーム
ローカルHTTPサーバー
Native Messaging
```

推奨は Native Messaging。

理由は以下。

- Chrome拡張とデスクトップアプリの連携に適している
- URLや選択テキストを安全に渡しやすい
- 将来的に拡張しやすい

ただし、実装難易度が高い場合は、v2.0ではローカルHTTPサーバー方式でもよい。

## 10. タスク詳細画面仕様

### 10.1 目的

タスクの補足情報を確認・編集できるようにする。

### 10.2 表示項目

```text
タイトル：
営業資料の修正版を作成してください

状態：
未完了

作業時間：
未入力

元アプリ：
Google Chrome

元URL：
https://example.com/message/123

元テキスト：
来週までに営業資料の修正版を作成してください。
特に価格表と導入事例を更新してください。

メモ：
[ 任意メモ欄 ]
```

### 10.3 操作

- タイトル編集
- メモ編集
- 元URLを開く
- タスク削除
- 完了状態変更
- source_text のコピー

### 10.4 元URLを開く

source_url が存在する場合、ボタンを表示する。

```text
[元ページを開く]
```

クリックすると、既定のブラウザでURLを開く。

## 11. 日報・1on1メモ連携仕様

### 11.1 目的

完了タスクを日報や1on1メモに貼り付けやすい形で出力する。

### 11.2 日報形式

```markdown
## 2026/05/28 作業ログ

### 完了したこと

- 仕様書を確認する：30分
- 営業資料の修正版を作成：1時間
- 会議メモ整理：20分

### 合計

- 完了件数：3件
- 作業時間：1時間50分
```

### 11.3 1on1メモ形式

```markdown
## 1on1メモ用 作業振り返り

### 今週やったこと

- 仕様書を確認する
- 営業資料の修正版を作成
- 会議メモ整理

### 時間がかかった作業

- 営業資料の修正版を作成：1時間

### 相談したいこと

- 未入力
```

### 11.4 操作

以下のボタンを追加する。

```text
[日報コピー]
[1on1メモコピー]
[Markdown出力]
```

### 11.5 週次1on1出力

v2では、過去7日間の完了タスクをまとめて1on1用メモとして出力できるようにする。

## 12. インポート・エクスポート仕様

### 12.1 目的

データ移行やバックアップを可能にする。

### 12.2 エクスポート形式

| 形式 | 用途 |
|---|---|
| JSON | バックアップ・復元 |
| CSV | Excel分析 |
| Markdown | 日報・メモ |
| SQLite | アプリ内部保存 |

### 12.3 CSV出力項目

```csv
id,title,completed,spent_minutes,created_at,completed_at,source_app,source_url
```

### 12.4 JSON出力例

```json
{
  "exported_at": "2026-05-28T12:00:00+09:00",
  "tasks": [
    {
      "id": "task_001",
      "title": "仕様書を確認する",
      "completed": true,
      "spent_minutes": 30,
      "created_at": "2026-05-28T09:00:00+09:00",
      "completed_at": "2026-05-28T09:30:00+09:00"
    }
  ]
}
```

### 12.5 インポート仕様

JSON形式のバックアップファイルからタスクを復元できるようにする。

インポート時は以下を行う。

- ファイル形式を検証する
- 既存IDと重複する場合は新しいIDを発行する
- インポート結果を表示する
- 失敗したタスクがある場合はエラー内容を表示する

## 13. v2で追加するデータ項目

| 項目 | 型 | 必須 | 説明 |
|---|---|---|---|
| source_text | string | 任意 | 選択元文章 |
| source_app | string | 任意 | 元アプリ名 |
| source_url | string | 任意 | 元URL |
| source_title | string | 任意 | 元ページタイトル |
| source_type | string | 任意 | web / app / clipboard |
| source_created_at | datetime | 任意 | タスク化日時 |
| tags | array | 任意 | タグ |
| exported_at | datetime | 任意 | エクスポート日時 |

## 15. セキュリティ・プライバシー方針

### 15.1 基本方針

v2では、選択文章やURLを保存するため、プライバシーに配慮する。

### 15.2 保存場所

- データは原則ローカル保存
- 外部サーバーへ送信しない
- ブラウザ拡張から送信する先はローカルアプリのみ

### 15.3 ユーザー確認

初回利用時に以下を説明する。

```text
選択した文章やURLは、タスク作成のためローカルに保存されます。
外部サーバーには送信されません。
```

### 15.4 削除

ユーザーは source_text や source_url を削除できるようにする。

## 16. 非対象機能

v2では以下を実装しない。

- チーム共有
- クラウド同期
- 権限管理
- コメントスレッド
- チャット機能
- AIによる自動タスク生成
- 外部カレンダー同期
- 複雑なプロジェクト管理

## 17. v2 完了条件

v2は、以下を満たした時点で完成とする。

- 選択文章をタスクに追加できる
- 右クリックメニューからタスク追加できる
- クリップボードからタスク追加できる
- 元文章を保存できる
- 元URLを保存できる
- 元アプリ名を保存できる
- タスク詳細画面で元情報を確認できる
- 元URLをブラウザで開ける
- 日報形式でMarkdownコピーできる
- 1on1メモ形式でMarkdownコピーできる
- JSON/CSV/Markdown形式でエクスポートできる
- JSON形式からインポートできる

## 18. v2の価値

v2により、本アプリは単なるToDoアプリではなく、作業中の情報をタスク化するタスクキャッチャーになる。

特に以下の価値が生まれる。

```text
・メール本文からタスクを作れる
・Webページの文章からタスクを作れる
・チャットの依頼文をタスク化できる
・元URLや元文章を後から確認できる
・日報や1on1メモに作業履歴を活用できる
```

## 19. v2開発順序

推奨開発順序は以下。

```text
1. クリップボードから追加
2. source_text保存
3. タスク詳細画面
4. source_url / source_app保存
5. ブラウザ拡張作成
6. 右クリックメニュー連携
7. 日報・1on1メモ出力強化
8. JSON/CSV/Markdownエクスポート
9. JSONインポート
```

まずはクリップボードから追加を実装し、その後にブラウザ拡張や右クリック連携へ進むのが安全である。
